01Goal & constraints

  • An MVP prediction market where users bet on YES/NO outcomes of future events with on-chain USDC. AMM pricing, owner-resolved outcomes, realtime updates. Production-ready, clean, extensible.
  • All TypeScript, strict. Sensitive config via env only. Every on-chain call has error handling + gas-safe patterns. README with local + deploy instructions.
  • Blockchain is the single source of truth for markets, pools, share balances, trades (as events) and outcomes. PostgreSQL is a rebuildable cache — reconstructable purely from chain (and the app self-heals it on startup).
  • Designed for a white-label v2 (one codebase, per-tenant via env), so keep all config env-driven.

04Smart contracts

Three contracts. AMMPricer = pure library, all functions internal pure (inlined → no deploy/link). MockUSDC = 6-dec ERC20 with open mint(to,amt) + faucet() (testnet only). PredictionMarket = the core.

AMMPricer — constant-product, correct direction

getYesPrice(yesPool, noPool) -> (yesPool * 1e18) / (yesPool + noPool)   // OWN pool / total
getNoPrice (yesPool, noPool) -> (noPool  * 1e18) / (yesPool + noPool)
getShares(yesPool, noPool, usdcAmount, isYes) -> usdcAmount * total / ownPool   // = amount / price
getPriceAfterBuy(yesPool, noPool, usdcAmount, isYes) -> price of bought side after amount enters its pool
assertInvariant(yesPool, noPool) -> require total>0; yesPrice+noPrice within [1e18-1, 1e18]
⚠ Direction matters. Buying a side adds to its OWN pool and RAISES its price (NO falls) — like Polymarket. Price = ownPool/total (NOT oppositePool/total, which inverts it). Keep lib/amm.ts (JS replica for instant UI preview) identical.

Worked example: seed 1000 → pools 500/500 → YES 50%. Buy YES 100 (fee 0) → shares = 100·1000/500 = 200; pools 600/500 → YES = 600/1100 = 54.5% (rises). Resolve YES, totalPool 1100, you hold 200 of totalYesShares → payout = 200/totalYes · 1100.

Functions (all require()s)

FunctionGuards / behavior
constructor(usdc, treasury, feeBps)Ownable(msg.sender); usdc≠0, treasury≠0, feeBps≤500.
createMarket(question, endTime, seedAmount) onlyOwner → idendTime>now, seedAmount>0; id=++marketCount; yesPool=seed/2, noPool=seed−seed/2; safeTransferFrom seed; emit MarketCreated. (Seed mints no shares.)
buyShares(id, isYes, usdcAmount, minShares) nonReentrant → sharesmarket exists, !resolved, now<endTime, amount>0; fee=amount·feeBps/10000 (→ accumulatedFees, NOT pool); net=amount−fee; shares=getShares(net); require shares≥minShares && shares>0; add net to side pool, mint shares, totalShares+=; safeTransferFrom full amount; emit SharesBought.
resolveMarket(id, outcome) onlyOwnerexists, now≥endTime, !resolved; set resolved+outcome; emit MarketResolved.
claimWinnings(id) nonReentrant → payoutresolved; userShares = winning-side shares; require >0; payout = userShares · totalPool / totalWinningShares; zero the claimer's winning shares (pool/totals untouched); safeTransfer; emit WinningsClaimed. Pull model, claimable forever, no admin sweep.
setFeeBps(n) onlyOwnern≤500.
setTreasury(a) onlyOwnera≠0.
withdrawFees() nonReentrantmsg.sender==treasury; amount=accumulatedFees>0; reset; safeTransfer.

05Contract tests (Hardhat, ~35, all pass)

Use AMMPricerHarness.sol to call the internal library externally. Use loadFixture + time helpers; cast deployed contracts via typechain.

  • AMMPricer: equal pools → 0.5; invariant across ratios; extreme ratios (tiny pool → that side cheap); getShares at 50/50 (=200) and imbalanced; getPriceAfterBuy raises bought side & bigger buy → bigger move; assertInvariant passes/reverts on empty.
  • PredictionMarket: createMarket owner/seed/emit + reverts (not-owner via revertedWithCustomError(market,'OwnableUnauthorizedAccount'), past endTime, zero seed); buyShares happy + fee (net in pool, fee accrues) + feeBps=0 + slippage pass; reverts (expired, resolved, zero, slippage, no-allowance via ERC20InsufficientAllowance); resolve YES/NO + reverts; claim pro-rata (read share counts from chain, don't hardcode) + loser/double-claim revert + not-resolved revert; fee mgmt (owner-only, cap 500, treasury-only withdraw, accrual); full lifecycle integration draining the contract to 0.

06Slippage & error handling

  • minShares = expectedShares * 0.99 (1% tolerance), passed into buyShares.
  • Warn if price impact |after−before|/before > 5%.
  • Friendly error map (parse nested viem shortMessage/details/message/cause): user rejected/4001, slippage→"Price moved too much", market expired→"This market has closed", already resolved, not resolved, no winning shares, insufficient allowance→"Approve USDC", insufficient balance, ownable→"Only the admin", fallback "Transaction failed: " + shortMessage.

07Database schema (Prisma · PostgreSQL · cache only)

model User     { address String @id  role String @default("TRADER")  createdAt DateTime @default(now())  trades Trade[] }
model Category { id String @id @default(cuid())  name String @unique  slug String @unique  icon String
                 isDefault Boolean @default(false)  createdAt DateTime  markets Market[] }
model Market   { id String @id                       // = String(contractId)
                 contractId Int @unique               // on-chain id
                 question String  description String  categoryId String  category Category @relation(...)
                 endDate DateTime  resolved Boolean @default(false)  outcome String?    // "YES"|"NO"|null
                 status String @default("ACTIVE")      // ACTIVE|EXPIRED|RESOLVED
                 imageUrl String?  yesLabel String @default("YES")  noLabel String @default("NO")
                 yesImageUrl String?  noImageUrl String?            // optional per-outcome images
                 yesPool Float @default(0)  noPool Float @default(0)  volume Float @default(0)   // cached chain state
                 createdAt DateTime  trades Trade[]   @@index([status]) @@index([categoryId]) @@index([endDate]) }
model Trade    { id String @id @default(cuid())  marketId String  market Market @relation(...)
                 trader String  user User? @relation(fields:[trader],references:[address])
                 outcome String  amount Float  shares Float  txHash String @unique  createdAt DateTime
                 @@index([marketId]) @@index([trader]) }
⚠ Added beyond the original spec (needed): cached yesPool/noPool/volume (for listing + sort-by-volume), yesImageUrl/noImageUrl, and txHash @unique (idempotent event replay). Prices are derived from pools in a serializer, never stored. binaryTargets = ["native","debian-openssl-3.0.x"] on the generator (Docker).
USDC = 6 decimals everywhere. Expose to the UI as a human number; convert at the chain boundary only.

08Data strategy & self-healing

On-chain (truth): all markets (active+resolved, never deleted), pools, share balances, trades (events), outcomes. Off-chain (DB cache): metadata (description, category, image, labels, outcome images), trades mirrored from events, indexed for fast queries.

Event listener (lib/eventListener.ts, viem watchContractEvent):

  • SharesBought → upsert User, update market pools + volume += amount, upsert Trade by txHash, emit market:trade.
  • MarketResolved → set resolved/outcome/status=RESOLVED, emit market:resolved.
  • MarketCreated → if a row exists (UI-created, has metadata) do nothing; else create a minimal row under "Other" and emit market:created.

Startup self-heal (run before watching):

  • reconcileMarketsFromChain() — read marketCount; for each on-chain market: update existing rows' chain fields (pools/resolved/outcome/status), or re-create missing rows from chain (on-chain question + default metadata).
  • backfillTradesFromChain() — scan SharesBought logs in chunks (SYNC_LOOKBACK_BLOCKS), upsert trades by txHash, recompute each market's volume from the Trade table.
⚠ Mandatory. Free hosts spin down and deploys restart the process; watchContractEvent only sees events from start-time, so without startup reconcile+backfill, markets/trades vanish from the UI after a sleep. Never casually wipe the DB — off-chain-only metadata (descriptions, custom labels, images) can't be rebuilt from chain.

10Realtime (Socket.io)

Rooms: market:{id} (detail), market:list (listing), admin (admins only, after signature verify). Auth: guests connect unauthenticated; Traders/Admins send { address, signature, message:"Connect to PredictX" }, verified via viem verifyMessage; admins additionally joined to admin.

market:trade            // -> market:{id} AND market:list
  { marketId, trader, isYes, outcomeLabel, usdcAmount, shares, yesPool, noPool, yesPrice, noPrice, volume, timestamp }
market:resolved         // -> market:{id}
  { marketId, outcome, outcomeLabel, totalPool, resolvedAt }
market:created          // -> market:list
  { marketId, question, description, category, endDate, yesLabel, noLabel, imageUrl, yesImageUrl, noImageUrl,
    yesPrice, noPrice, yesPool, noPool, volume, status:"ACTIVE", createdAt }
market:expired          // -> market:{id} AND market:list   { marketId, expiredAt }
admin:resolve:pending   // -> admin
  { marketId, question, endDate, yesPool, noPool, yesPrice, noPrice, yesLabel, noLabel, volume, imageUrl }

Client hooks: useMarketSocket(id) joins market:{id}, handles trade/resolved/expired. useMarketListSocket() joins market:list → marketStore (applyTrade/prependCreated/setExpired). useAdminSocket() (Header) → adminStore on admin:resolve:pending. Connect manually after wallet auth; cache the signature in sessionStorage; never auto-open the connect modal.

⚠ Hold the Socket.io server on globalThis (e.g. globalThis.__predictxIO): server.ts (tsx) and Next-bundled API routes (webpack) are separate module graphs, so a plain singleton diverges and route emits silently no-op.

11Cron + client-side effective status

node-cron('* * * * *') in server.ts: query ACTIVE markets with endDate ≤ now → set EXPIRED; emit market:expired (market+list) and admin:resolve:pending (admin). Boot order: load env → prepare Next → HTTP server → attach Socket.io → cron → event listener → listen.

⚠ Cron fires only on the minute (≤60s lag). Add useEffectiveStatus(market): returns RESOLVED if resolved, else EXPIRED if endDate ≤ now, else ACTIVE — and schedules a setTimeout that forces a re-render exactly at endDate. Cards / detail / TradingPanel use it so a market closes the instant the countdown hits zero, regardless of the cron.

14Edit market — integrity locks (/market/[id]/edit, admin)

Admin edits off-chain metadata, but a market's terms must not change under traders.

FieldRule
questionNever editable — on-chain commitment (immutable). Shown read-only.
description, yes/no labelsEditable only before any trade; locked once totalShares > 0 on-chain.

15Images — data URIs (no filesystem)

⚠ Never write uploads to public/uploads. On ephemeral hosts the disk is wiped on restart and runtime-written public files don't serve reliably → broken images. POST /api/upload (admin): validate type/≤2MB → sharp resize ~256×256 webp → return data:image/webp;base64,… stored directly in the DB. Fall back to original bytes if sharp throws. (Swap to S3/R2 only at high volume — then imageUrl becomes a real URL; data URIs bloat list responses.)

Per-outcome yesImageUrl/noImageUrl are off-chain (no contract change); shown in the probability bar (replacing the dot), the segmented trading toggle, the detail price blocks, and live-created cards.

18USDC approval flow (useTrade)

1 read allowance(user, CONTRACT)
2 if < amount: approve(CONTRACT, MaxUint256) and wait  (status 'approving')   // approve once ever
3 minShares = expectedShares * 0.99 ; buyShares(id, isYes, amount, minShares) and wait  (status 'buying')
4 status 'confirmed' ; refetch balances
UI: "Approve USDC" -> "Approving…" -> "Buy YES/NO" -> "Buying…" -> "Confirmed ✓". Admin also approves before createMarket.

20White-label & runtime config

One codebase per tenant via env. tenant.config.ts reads TENANT_*; the root layout injects --color-primary/yes/no as CSS variables on <html> and passes the tenant object to a client provider. Rule: zero hardcoded hex in components — always var(--color-…) tokens. Differs per tenant: CONTRACT/ADMIN/TREASURY/FEE_BPS, DATABASE_URL, RPC, chain id, TENANT_*, domain. Same: all code, contracts source, schema, socket events.

⚠ Resolve NEXT_PUBLIC_* at RUNTIME, not build time. Some hosts (Render) don't pass env to the Docker build, so build-inlined publics come out empty. Root layout (server, force-dynamic) reads process.env per request and injects window.__PREDICTX_ENV__ via an inline script BEFORE the bundles run; lib/constants reads window on the client, process.env on the server. Bonus: change config = restart, no rebuild.

Future protocol (v2): a Factory contract — clients deploy their own market contract (they pay gas, they own it) with an immutable protocol fee → your address on every trade; one static multi-tenant frontend reads everything from chain + IPFS. Self-serve, hands-off, you collect the protocol fee. (Mind prediction-market regulation.)

22Deployment (Docker → free host now, VPS later)

Single Dockerfile (node:22-bookworm-slim): apt-get install openssl ca-certificates → npm ci → copy → declare NEXT_PUBLIC_* ARGs → npm run compile (ABIs) → prisma generate → next build. Entrypoint: prisma db push → (if RESET_DB=true wipe markets+trades) → seed categories → run server.ts. docker-compose.yml (app + Postgres) for any VPS.

  • Free stack: Render (Docker Web Service) + Render/Neon Postgres + a public testnet. Render free spins down ~15 min — startup self-heal (§08) covers it.
  • Flow: deploy contracts (deploy:<net>, SEED_MARKETS=false) → copy addresses → push repo → Render: Docker + env vars → deploy → admin creates markets via UI.
⚠ Prisma on slim: without openssl in the image + binaryTargets debian-openssl-3.0.x → "Schema engine error" on db push.
⚠ RESET_DB is one-shot: set true for one deploy after a contract redeploy, then REMOVE it — else every spin-down wipes admin-created markets.

Switching chains (e.g. Polygon) = env-only + ~2 lines to register a brand-new chain in lib/chains.ts + hardhat.config.ts: redeploy contracts, update CHAIN_ID/RPC/addresses (real USDC on mainnet, MockUSDC on testnet), reset DB. App/contract code never changes; USDC is 6-dec on all chains.

25Gotchas checklist ⚠ (don't repeat these)

  • AMM direction: price = ownPool/total; buying a side raises its own price. Keep Solidity + lib/amm.ts identical.
  • Cache columns: add yesPool/noPool/volume to Market; derive prices in a serializer.
  • Self-heal on startup: reconcile markets + backfill trades from chain (free hosts wipe/sleep).
  • Socket via globalThis (tsx server vs Next webpack bundle).
  • Effective status on client (cron lags expiry up to 60s).
  • Images as data URIs in DB (no ephemeral filesystem).
  • Edit integrity locks: question immutable; description/labels lock once trading starts (server-enforced 409).
  • wagmi isReconnecting guard in useRole/RoleGate.
  • Runtime NEXT_PUBLIC injection (hosts may not pass env to the Docker build).
  • color-mix, not Tailwind /alpha, on CSS-variable colors.
  • Prisma on slim: install openssl + set binaryTargets debian-openssl-3.0.x.
  • RESET_DB one-shot ; SEED_MARKETS=false on shared testnets ; EXISTING_USDC to keep balances across redeploys.
  • loadEnv first in server.ts ; accept private keys with/without 0x ; lucide-react is legitimately 1.x.
